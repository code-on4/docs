N-Gate is a multi-tenant API Gateway, Infrastructure, and Integration Platform designed for institutions that need a secure bridge between their services. It lets you manage workspaces and teams, proxy API traffic to backend services, administer connected gateways and consumers, process business mandates, and coordinate physical or virtual remote infrastructure agents — all through a unified API secured by Bearer token authentication.

## Getting Started

<Steps>
  <Step title="Authenticate">
    Call `POST /auth/login` with your credentials to receive a Bearer token. Include it in the `Authorization` header on every subsequent request.
  </Step>
  <Step title="Create a Workspace">
    Use `POST /api/workspaces` to provision a new workspace for your institution. Workspaces are the root container for all resources in N-Gate.
  </Step>
  <Step title="Configure Your First Route">
    Define a proxy route with `POST /api/routes`, pointing N-Gate at your target backend service. Then call `POST /api/routes/proxy` to forward requests through it.
  </Step>
  <Step title="Invite Your Team">
    Use `POST /api/workspaces/:workspaceId/invite` to bring team members into your workspace and assign them Permission Roles.
  </Step>
</Steps>

<Note>
  All N-Gate API endpoints except `/auth/login` require a valid Bearer token. See the [Authentication guide](/authentication) for details.
</Note>
