The Airtime Transactions API gives you read access to every airtime top-up event processed through N-Gate's telecom integration layer. Transactions are created when a downstream service initiates an airtime disbursement and move through `pending → completed | failed` as the upstream provider confirms or rejects the transfer. You can filter the list endpoint by provider, date window, and status to build dashboards, reconcile records, or trigger downstream workflows when a transaction settles.

---

## List Airtime Transactions

<br />

```http
GET /api/airtime-transactions
```

Returns a paginated list of airtime transactions. All query parameters are optional; omitting them returns the full history the caller has access to, ordered by `createdAt` descending.

### Query Parameters

<ParamField query="workspaceId" type="string">
  Scope results to a single workspace. Recommended for multi-tenant contexts to
  avoid unintentionally returning cross-workspace records.
</ParamField>

<ParamField query="status" type="string">
  Filter by transaction state. One of:
  - `pending` — the top-up request has been submitted but not yet confirmed.
  - `completed` — the provider confirmed successful credit delivery.
  - `failed` — the provider rejected or could not process the request.
</ParamField>

<ParamField query="provider" type="string">
  Filter by telecom provider slug (e.g. `"mtn"`, `"airtel"`, `"glo"`). Provider
  slugs are listed in your workspace's provider configuration.
</ParamField>

<ParamField query="startDate" type="string">
  ISO 8601 date-time. Return only transactions created at or after this
  timestamp (e.g. `"2024-11-01T00:00:00.000Z"`).
</ParamField>

<ParamField query="endDate" type="string">
  ISO 8601 date-time. Return only transactions created at or before this
  timestamp (e.g. `"2024-11-30T23:59:59.999Z"`). Must be greater than
  `startDate` when both are provided.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number to retrieve. Starts at `1`.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Number of records per page. Maximum is `100`.
</ParamField>

### Response

<ResponseField name="data" type="array">
  <Expandable title="Transaction object">
    <ResponseField name="id" type="string">
      Unique transaction identifier.
    </ResponseField>
    <ResponseField name="amount" type="number">
      Top-up amount in the smallest unit of `currency` (e.g. kobo for NGN).
    </ResponseField>
    <ResponseField name="currency" type="string">
      ISO 4217 currency code (e.g. `"NGN"`, `"KES"`, `"GHS"`).
    </ResponseField>
    <ResponseField name="recipient" type="string">
      Phone number that received the airtime credit, in E.164 format.
    </ResponseField>
    <ResponseField name="provider" type="string">
      Telecom provider slug used for this transaction.
    </ResponseField>
    <ResponseField name="status" type="string">
      Current transaction state: `pending`, `completed`, or `failed`.
    </ResponseField>
    <ResponseField name="workspaceId" type="string">
      Workspace that initiated the transaction.
    </ResponseField>
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp of when the transaction was initiated.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  <Expandable title="Pagination object">
    <ResponseField name="page" type="integer">Current page number.</ResponseField>
    <ResponseField name="limit" type="integer">Records per page.</ResponseField>
    <ResponseField name="total" type="integer">Total number of matching records.</ResponseField>
  </Expandable>
</ResponseField>

<CodeGroup>
```bash cURL — filtered by status and date range
curl --request GET \
  --url "https://api.n-gate.io/api/airtime-transactions?workspaceId=ws_01&status=completed&startDate=2024-11-01T00:00:00.000Z&endDate=2024-11-30T23:59:59.999Z&page=1&limit=20" \
  --header "Authorization: Bearer <token>"
```

```bash cURL — filter by provider
curl --request GET \
  --url "https://api.n-gate.io/api/airtime-transactions?workspaceId=ws_01&provider=mtn" \
  --header "Authorization: Bearer <token>"
```

## Get a Single Transaction

<br />

```http
GET /api/airtime-transactions/:id
```

Retrieves the full detail record for one airtime transaction. For failed transactions, the response includes a `failureReason` field explaining why the provider rejected the request, which is useful for support investigations and automated retry logic.

### Path Parameters

<ParamField path="id" type="string" required>
  The unique ID of the airtime transaction to retrieve.
</ParamField>

### Response

<ResponseField name="id" type="string">Unique transaction identifier.</ResponseField>
<ResponseField name="amount" type="number">
  Top-up amount in the smallest unit of `currency`.
</ResponseField>
<ResponseField name="currency" type="string">ISO 4217 currency code.</ResponseField>
<ResponseField name="recipient" type="string">Recipient phone number in E.164 format.</ResponseField>
<ResponseField name="provider" type="string">Telecom provider slug.</ResponseField>
<ResponseField name="status" type="string">
  Current state: `pending`, `completed`, or `failed`.
</ResponseField>
<ResponseField name="workspaceId" type="string">Owning workspace.</ResponseField>
<ResponseField name="failureReason" type="string">
  Present only when `status` is `"failed"`. A short string from the provider
  describing the rejection reason (e.g. `"INVALID_MSISDN"`, `"INSUFFICIENT_BALANCE"`).
</ResponseField>
<ResponseField name="createdAt" type="string">ISO 8601 initiation timestamp.</ResponseField>

<Note>
  `failureReason` is omitted entirely from the response object when a
  transaction has not failed. Check the `status` field before reading this
  property in your integration code.
</Note>

<CodeGroup>
```bash cURL — successful transaction
curl --request GET \
  --url "https://api.n-gate.io/api/airtime-transactions/txn_air001" \
  --header "Authorization: Bearer <token>"
```

```bash cURL — failed transaction

```

## Error Responses

All errors follow the standard N-Gate error envelope:

```json
{
  "error": "<message>",
  "statusCode": 400,
  "timestamp": "2024-11-15T12:05:00.000Z"
}
```

| Status | Meaning                                                |
| ------ | ------------------------------------------------------ |
| `400`  | Invalid query parameters (e.g. `endDate` before `startDate`). |
| `401`  | Missing or invalid Bearer token.                       |
| `403`  | Token lacks transaction-read permission.               |
| `404`  | Transaction not found.                                 |
