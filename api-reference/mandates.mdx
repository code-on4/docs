---

The Mandates API lets you programmatically submit business mandate documents — such as regulatory authorizations, partner agreements, or compliance certificates — for review and track them through the approval lifecycle. Each mandate is associated with a workspace, given a type classification, and moves through `pending → approved | rejected` states as your operations team acts on it. Documents are stored securely and a hosted URL is returned so you can reference them later without re-uploading.

---

## Submit a Mandate

<br />

```http
POST /api/mandates
```

Uploads a mandate document and opens a new mandate record in `pending` status. The request must use `multipart/form-data` encoding — standard JSON bodies are not accepted for this endpoint because the document binary must be included in the same request.

<Note>
  Set `Content-Type: multipart/form-data` when making this request. Most HTTP
  clients (cURL with `--form`, Axios with `FormData`, Postman's form-data mode)
  handle the boundary header automatically — do not set the boundary manually.
</Note>

### Body Parameters

<ParamField body="document" type="file" required>
  The mandate document to upload. Accepted formats: PDF, DOCX, PNG, JPEG.
  Maximum file size is 10 MB.
</ParamField>

<ParamField body="workspaceId" type="string" required>
  The workspace this mandate is submitted under.
</ParamField>

<ParamField body="type" type="string" required>
  Classification of the mandate (e.g. `"regulatory"`, `"partner-agreement"`,
  `"compliance-certificate"`). Allowed values are defined per workspace
  configuration.
</ParamField>

<ParamField body="description" type="string">
  An optional free-text description providing context for reviewers.
</ParamField>

### Response

<ResponseField name="id" type="string">Unique ID assigned to the new mandate.</ResponseField>
<ResponseField name="workspaceId" type="string">Workspace the mandate belongs to.</ResponseField>
<ResponseField name="type" type="string">Mandate type classification.</ResponseField>
<ResponseField name="description" type="string">Description, if provided.</ResponseField>
<ResponseField name="status" type="string">Always `"pending"` immediately after submission.</ResponseField>
<ResponseField name="documentUrl" type="string">
  Signed URL to the uploaded document. Valid for 24 hours; request fresh details
  via `GET /api/mandates/:id` to obtain a new URL after expiry.
</ResponseField>
<ResponseField name="createdAt" type="string">ISO 8601 submission timestamp.</ResponseField>

<CodeGroup>
```bash cURL
curl --request POST \
  --url "https://api.n-gate.io/api/mandates" \
  --header "Authorization: Bearer <token>" \
  --form "document=@/path/to/mandate.pdf" \
  --form "workspaceId=ws_01" \
  --form "type=regulatory" \
  --form "description=Annual AML compliance certificate for Q4 2024"
```

```json Response
{
  "id": "mnd_def456",
  "workspaceId": "ws_01",
  "type": "regulatory",
  "description": "Annual AML compliance certificate for Q4 2024",
  "status": "pending",
  "documentUrl": "https://storage.n-gate.io/mandates/mnd_def456/mandate.pdf?token=...",
  "createdAt": "2024-11-17T14:00:00.000Z"
}
```
</CodeGroup>

## List Mandates

<br />

```http
GET /api/mandates
```

Returns a paginated list of mandates. Use `status` to filter by approval state and `workspaceId` to scope results to a single workspace.

### Query Parameters

<ParamField query="workspaceId" type="string">
  Return only mandates belonging to this workspace.
</ParamField>

<ParamField query="status" type="string">
  Filter by approval status. One of `pending`, `approved`, `rejected`.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number to retrieve. Starts at `1`.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Number of records per page. Maximum is `100`.
</ParamField>

### Response

<ResponseField name="data" type="array">
  <Expandable title="Mandate summary object">
    <ResponseField name="id" type="string">
      Unique mandate identifier.
    </ResponseField>
    <ResponseField name="type" type="string">
      Mandate type classification.
    </ResponseField>
    <ResponseField name="status" type="string">
      Current approval status: `pending`, `approved`, or `rejected`.
    </ResponseField>
    <ResponseField name="createdAt" type="string">
      ISO 8601 submission timestamp.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  <Expandable title="Pagination object">
    <ResponseField name="page" type="integer">Current page number.</ResponseField>
    <ResponseField name="limit" type="integer">Records per page.</ResponseField>
    <ResponseField name="total" type="integer">Total number of matching records.</ResponseField>
  </Expandable>
</ResponseField>

<CodeGroup>
```bash cURL
curl --request GET \
  --url "https://api.n-gate.io/api/mandates?workspaceId=ws_01&status=pending&page=1&limit=20" \
  --header "Authorization: Bearer <token>"
```

```json Response
{
  "data": [
    {
      "id": "mnd_def456",
      "type": "regulatory",
      "status": "pending",
      "createdAt": "2024-11-17T14:00:00.000Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 1
  }
}
```
</CodeGroup>

## Get Mandate Details

<br />

```http
GET /api/mandates/:id
```

Fetches the full detail record for a single mandate, including the refreshed `documentUrl` and any reviewer-set timestamps.

### Path Parameters

<ParamField path="id" type="string" required>
  The unique ID of the mandate to retrieve.
</ParamField>

### Response

<ResponseField name="id" type="string">Unique mandate identifier.</ResponseField>
<ResponseField name="workspaceId" type="string">Owning workspace.</ResponseField>
<ResponseField name="type" type="string">Mandate type classification.</ResponseField>
<ResponseField name="description" type="string">Optional description.</ResponseField>
<ResponseField name="status" type="string">Current approval status.</ResponseField>
<ResponseField name="documentUrl" type="string">
  Fresh signed URL to the mandate document, valid for 24 hours.
</ResponseField>
<ResponseField name="createdAt" type="string">ISO 8601 submission timestamp.</ResponseField>
<ResponseField name="updatedAt" type="string">
  ISO 8601 timestamp of the last status change (e.g. when a reviewer approved or rejected the mandate).
</ResponseField>

<CodeGroup>
```bash cURL
curl --request GET \
  --url "https://api.n-gate.io/api/mandates/mnd_def456" \
  --header "Authorization: Bearer <token>"
```

```json Response
{
  "id": "mnd_def456",
  "workspaceId": "ws_01",
  "type": "regulatory",
  "description": "Annual AML compliance certificate for Q4 2024",
  "status": "approved",
  "documentUrl": "https://storage.n-gate.io/mandates/mnd_def456/mandate.pdf?token=...",
  "createdAt": "2024-11-17T14:00:00.000Z",
  "updatedAt": "2024-11-18T09:15:00.000Z"
}
```
</CodeGroup>

## Error Responses

All errors follow the standard N-Gate error envelope:

```json
{
  "error": "<message>",
  "statusCode": 400,
  "timestamp": "2024-11-17T14:05:00.000Z"
}
```

| Status | Meaning                                                          |
| ------ | ---------------------------------------------------------------- |
| `400`  | Missing required fields, unsupported file type, or file too large. |
| `401`  | Missing or invalid Bearer token.                                 |
| `403`  | Token lacks mandate-write permission.                            |
| `404`  | Mandate not found.                                               |
| `415`  | Request was not sent as `multipart/form-data`.                   |
