Workspaces are the foundational organizational unit in N-Gate. Every route, member, and permission belongs to a workspace, allowing teams to isolate their gateway configurations and access controls. The Workspaces API lets you list all workspaces you belong to, create new ones, and retrieve details about a specific workspace — including its current membership count.

---

## List Workspaces

<br />

`GET /api/workspaces`

Returns a paginated list of all workspaces the authenticated user is a member of.

### Query Parameters

<ParamField query="page" type="integer" default="1">
  The page number to retrieve. Defaults to `1`.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Number of workspaces to return per page. Defaults to `20`. Maximum is `100`.
</ParamField>

### Response

<ResponseField name="data" type="array">
  An array of workspace objects belonging to the authenticated user.
  <Expandable title="Workspace object">
    <ResponseField name="id" type="string">
      The unique identifier for the workspace.
    </ResponseField>
    <ResponseField name="name" type="string">
      The display name of the workspace.
    </ResponseField>
    <ResponseField name="description" type="string">
      An optional human-readable description of the workspace.
    </ResponseField>
    <ResponseField name="createdAt" type="string (ISO 8601)">
      Timestamp of when the workspace was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  Pagination metadata for the response.
  <Expandable title="Pagination object">
    <ResponseField name="page" type="integer">
      The current page number.
    </ResponseField>
    <ResponseField name="limit" type="integer">
      The number of records per page.
    </ResponseField>
    <ResponseField name="total" type="integer">
      The total number of workspaces across all pages.
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl --request GET \
  --url "https://api.n-gate.io/api/workspaces?page=1&limit=20" \
  --header "Authorization: Bearer <token>"
```

```json Response
{
  "data": [
    {
      "id": "ws_01hx9zt3m5kbf2qp",
      "name": "Payments Team",
      "description": "Workspace for the payments integration squad.",
      "createdAt": "2024-03-15T10:22:00Z"
    },
    {
      "id": "ws_01hx9zt9r7ncd4lw",
      "name": "Analytics Platform",
      "description": null,
      "createdAt": "2024-04-01T08:00:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 2
  }
}
```
</CodeGroup>

## Create a Workspace

<br />

`POST /api/workspaces`

Creates a new workspace and automatically adds the authenticated user as its owner.

### Request Body

<ParamField body="name" type="string" required>
  The name of the workspace. Must be unique within your account. Maximum 64 characters.
</ParamField>

<ParamField body="description" type="string">
  An optional description for the workspace. Maximum 255 characters.
</ParamField>

### Response

<ResponseField name="id" type="string">
  The unique identifier assigned to the newly created workspace.
</ResponseField>

<ResponseField name="name" type="string">
  The name provided for the workspace.
</ResponseField>

<ResponseField name="description" type="string">
  The description provided, or `null` if not supplied.
</ResponseField>

<ResponseField name="createdAt" type="string (ISO 8601)">
  Timestamp of when the workspace was created.
</ResponseField>

<Note>
  The authenticated user is automatically assigned the **Owner** role in the new workspace. Owner permissions cannot be revoked through the Permissions API.
</Note>

### Example

<CodeGroup>
```bash cURL
curl --request POST \
  --url "https://api.n-gate.io/api/workspaces" \
  --header "Authorization: Bearer <token>" \
  --header "Content-Type: application/json" \
  --data '{
    "name": "Payments Team",
    "description": "Workspace for the payments integration squad."
  }'
```

```json Response
{
  "id": "ws_01hx9zt3m5kbf2qp",
  "name": "Payments Team",
  "description": "Workspace for the payments integration squad.",
  "createdAt": "2024-03-15T10:22:00Z"
}
```
</CodeGroup>

## Get a Workspace

<br />

`GET /api/workspaces/:workspaceId`

Retrieves full details for a single workspace, including its current member count.

### Path Parameters

<ParamField path="workspaceId" type="string" required>
  The unique identifier of the workspace to retrieve.
</ParamField>

### Response

<ResponseField name="id" type="string">
  The unique identifier for the workspace.
</ResponseField>

<ResponseField name="name" type="string">
  The display name of the workspace.
</ResponseField>

<ResponseField name="description" type="string">
  The workspace description, or `null` if not set.
</ResponseField>

<ResponseField name="memberCount" type="integer">
  The total number of members currently in the workspace, including the owner.
</ResponseField>

<ResponseField name="createdAt" type="string (ISO 8601)">
  Timestamp of when the workspace was created.
</ResponseField>

<Warning>
  You will receive a `403 Forbidden` if you attempt to fetch a workspace you are not a member of.
</Warning>

### Example

<CodeGroup>
```bash cURL
curl --request GET \
  --url "https://api.n-gate.io/api/workspaces/ws_01hx9zt3m5kbf2qp" \
  --header "Authorization: Bearer <token>"
```

```json Response
{
  "id": "ws_01hx9zt3m5kbf2qp",
  "name": "Payments Team",
  "description": "Workspace for the payments integration squad.",
  "memberCount": 7,
  "createdAt": "2024-03-15T10:22:00Z"
}
```
</CodeGroup>

## Error Responses

All Workspaces API errors follow the standard N-Gate error shape:

```json
{
  "error": "Workspace not found",
  "statusCode": 404,
  "timestamp": "2024-06-01T12:00:00Z"
}
```

| Status Code | Meaning                                              |
|-------------|------------------------------------------------------|
| `400`       | Invalid request body (e.g. missing `name`)           |
| `401`       | Missing or invalid `Authorization` token             |
| `403`       | Authenticated user is not a member of the workspace  |
| `404`       | Workspace with the given `workspaceId` does not exist|
| `409`       | A workspace with that name already exists            |
